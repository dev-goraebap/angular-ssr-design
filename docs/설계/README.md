## 다음 단계

설계가 모두 확정되었으므로 구현으로 넘어갈 수 있다. 권장 착수 순서:

1. **shared/ui 순수 UI 구축** — spartan `init` + 컴포넌트 add, 적응형 `Sheet`(GSAP) ([ADR-0001](./ADR-0001-ui-기술스택.md), [ADR-0007](./ADR-0007-프로젝트-구조.md))
2. **백엔드 목 API** — `backend/` 별도 프로세스 + 인메모리 데이터 ([ADR-0003](./ADR-0003-백엔드-데이터-계층.md))
3. **인프라 배선** — SSR 프록시, `API_BASE_URL` 토큰, 인터셉터(baseURL·인증 포워딩), 라우트/렌더 모드 ([ADR-0002](./ADR-0002-렌더링-전략.md), [ADR-0004](./ADR-0004-인증.md))
4. **페이지 구현** — SSG/SSR/CSR 페이지, `httpResource` 데이터 페칭, 무한 스크롤 ([ADR-0005](./ADR-0005-데이터-페칭.md), [ADR-0006](./ADR-0006-목록-UX.md))
