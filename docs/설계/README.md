## 구현 현황 (✅ 완료)

ADR-0008의 7단계 계획대로 [`responsive-web-ux-strategy`](../../README.md)를 이식하고 SSR을 결합했다. 모든 단계가 구현·검증되었다.

| 단계 | 내용 | 관련 ADR |
|------|------|----------|
| 1 | SSR 인프라 + 렌더 모드 매핑 + 기존 코드 이식 | 0002, 0008 |
| 2 | 백엔드 목 API(`backend/`, 4001) + `/api` 프록시 | 0003 |
| 3 | 영화 읽기 → `httpResource`, SSR HTML에 데이터 포함 | 0005 |
| 4 | 인증 쿠키화 + 위시리스트/평점 백엔드 + Dexie 제거 | 0004 |
| 5 | 브라우저 가드, 모바일 LAN 접속 | 0001 |

### 구현하며 확인된 실측 노트

- **Angular 22 SSRF 방어가 실제로 발동**했다. `angular.json`의 `security.allowedHosts: []`(빌드)로 SSR 런타임이 모든 host 헤더를 400으로 거부 → `["localhost"]` 등 명시 필요(리서치 §5 실증).
- **SSR에 데이터가 채워지자 무한 스크롤 센티넬이 서버에서 렌더**되며 `IntersectionObserver`가 터졌다 → `isPlatformBrowser` 가드로 해결(ADR-0001 캡슐화 규칙). "데이터가 SSR에 들어왔다"는 방증.
- **dev와 프로덕션의 프록시 경로가 다르다.** 프로덕션은 `server.ts`의 `http-proxy-middleware`, dev(ng serve)는 그 미들웨어를 쓰지 않으므로 **`proxy.conf.json` + serve 옵션**으로 `/api`를 프록시한다.
- **ADR-0004의 SSR 쿠키 포워딩 인터셉터는 구현하지 않았다.** 개인화 화면(위시리스트·내 평점)이 모두 CSR(ADR-0002)이라, 인증 상태는 브라우저에서 `/api/me`로 확인하면 충분하다(불필요한 복잡도 회피, `auth.service.ts` 주석에 근거 명시).
- **SSG(홈)·빌드 타임 prerender는 백엔드 의존**: 빌드 시에도 4001이 떠 있어야 정적 HTML에 데이터가 포함된다.
