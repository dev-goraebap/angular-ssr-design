## 다음 단계

설계가 모두 확정되었다. 동일 도메인 CSR 구현([ADR-0008](./ADR-0008-기존-구현-이식.md))을 **이식**하면서 SSR을 결합하는 방향이므로, 착수 순서는 ADR-0008의 단계별 계획을 따른다:

1. **SSR 인프라 + 렌더 모드 매핑** ([ADR-0002](./ADR-0002-렌더링-전략.md)) — 스캐폴드는 이미 있음
2. **shared/ui·lib 이식** — 적응형 시트·GSAP·breakpoint·테마 + 브라우저 가드 ([ADR-0001](./ADR-0001-ui-기술스택.md))
3. **백엔드 목 API** — SEED 데이터를 `backend/`로 이전 ([ADR-0003](./ADR-0003-백엔드-데이터-계층.md))
4. **데이터 계층 전환** — Dexie(IndexedDB) → `httpResource` ([ADR-0005](./ADR-0005-데이터-페칭.md))
5. **인증 쿠키화** — localStorage → HttpOnly 쿠키 + SSR 포워딩 ([ADR-0004](./ADR-0004-인증.md))
6. **페이지 이식 + 렌더 모드 적용** — 무한 스크롤 포함 ([ADR-0006](./ADR-0006-목록-UX.md))
7. **검증** — 모달/뒤로가기/스크롤복원 + SSR 데이터 포함 + a11y
